# 8. 보안·품질·정책 부담

## 보안·품질·정책 관점에서 추가로 생기는 부담

모바일앱은 웹보다 **관리 범위가 넓습니다**.

- **OWASP MASVS**(모바일 보안 표준) 준수 (저장 데이터 보호, 인증·인가, 네트워크 통신 등)
- **행정안전부·KISA** 모바일 보안취약점 점검 가이드라인 준용
- **KRDS 디지털 정부서비스 UI/UX 가이드라인** (앱 전용 IA·화면흐름·네이티브 UI 별도 고려)
- **스토어 심사 대응**, OS 업데이트 대응, target API 유지, 메타데이터 관리, 장애 대응 체계 구축 필요

> **비용 경고**: 단순 화면 개발비 외에 **보안 설계·점검·릴리즈 검증 비용**이 추가됩니다.


## 금융권 하이브리드 앱 전용 보안 솔루션

금융권 하이브리드 앱의 보안성을 강화하기 위해 사용되는 주요 전용 보안 솔루션과 기술은 **인증**, **입력 보안**, **데이터 암호화**, **앱 위변조 방지** 등 여러 계층으로 구성됩니다.

### 1. 인증 및 사용자 식별 솔루션

| 솔루션/기술 | 설명 | 특징 |
|------------|------|------|
| **인증 통합 솔루션** | WizIN-Delfino G10 등 모바일 전용 인증 통합 솔루션 | 전자서명 및 사용자 인증 담당 |
| **바이오인증 (FIDO)** | 스마트폰 고유 하드웨어(지문·홍채·얼굴인식) 활용 생체인증 | 생체정보 분산 관리로 해킹 위험 감소, 표준 기술 적용 |
| **모바일 OTP 및 인증서** | 모피어스(Morpheus) 등 플랫폼 내 결합 제공 | OTP + 인증서 일원화 관리 |

### 2. 입력 보안 및 구간 암호화 솔루션

| 솔루션/기술 | 설명 | 특징 |
|------------|------|------|
| **키보드/가상 키패드 보안** | nProtect Online Security v1.0 등 | 입력 정보 유출 방지, 키보드 보안 강화 |
| **데이터 암호화 API** | 펜타시큐리티(Penta Security) 암호화/복호화 API | 정형·비정형 데이터 저장·전송 구간 보안 |

### 3. 앱 무결성 및 시스템 관리 솔루션

| 솔루션/기술 | 설명 | 특징 |
|------------|------|------|
| **앱 위변조 체크** | 전용 플랫폼을 통해 소스코드·리소스 임의 변경 여부 실시간 감시 | 하이브리드 앱은 웹 리소스 포함으로 필수 |
| **취약점 진단 솔루션** | 가디언 워치(Guardian Watch) 등 소스코드 보안 약점(PMD 점검)·시스템 취약점 상시 진단 | 개발 단계 및 운영 중 지속 |
| **MDM/MAM** | Mobile Device Management + Mobile Application Management | 관리 앱/비관리 앱 간 보안 균형 유지 |

### 4. 인프라 및 전송 보안

| 기술 | 설명 | 효과 |
|------|------|------|
| **API Gateway** | 외부 시스템과 안전 데이터 연동 | 인터페이스 구간 보안·인증 통합 관리 |
| **구간 암호화 (SSL)** | 모든 모바일 웹·앱 통신 구간에 SSL 인증서 적용 | 스니핑 등 전송 중 공격 차단 |

> **준수 요건**: 이러한 솔루션들은 **금융감독원의 모바일 앱 프로그램 운영관리 가이드라인**을 준수해야 하며, 하이브리드 앱은 **네이티브 영역과 웹 영역 간의 브릿지 보안**이 취약점이 될 수 있으므로 이에 대한 **정밀한 보안 심사(Vetting) 프로세스**가 병행되어야 합니다.
