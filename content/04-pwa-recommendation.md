# PWA 우선 검토 권고

> **권고사항**: PWA는 단일 코드베이스로 광범위한 기기에 도달하면서 홈화면 설치·푸시·오프라인 등 앱 유사 기능을 제공합니다. 플랫폼별 지원 편차만 확인하면 충분히 대안이 될 수 있습니다.

## PWA로 가능한 것

| 기능 | 쉬운 설명 | 실제 예시 |
|------|----------|----------|
| **홈화면 설치** | 웹사이트를 스마트폰 홈화면에 아이콘으로 저장<br>앱 스토어 설치 없이 아이콘 하나로 바로 접속 | 카카오톡 아이콘 누르듯, PWA 아이콘 탭 → 바로 사이트 진입 |
| **오프라인 캐싱** | 인터넷이 끊겨도 이전에 본 내용을 볼 수 있음<br>Service Worker가 데이터를 기기에 미리 저장 | 지하철에서 기사 미리 로드 → 오프라인에서도 읽기 가능 |
| **푸시 알림** | 브라우저에서 앱처럼 알림 메시지 발송<br>Android는 완전 지원, iOS는 제한적 | "주문이 배송 시작되었습니다" 알림 → 탭하면 주문 상세 화면 |
| **배경 동기화** | 인터넷 연결 회복 시 자동으로 데이터 전송<br>오프라인에서 작성한 내용을 연결 시 자동 업로드 | 오프라인에서 장바구니 담음 → WiFi 연결 시 자동 서버 동기화 |
| **앱 스타일 UI** | 네이티브 앱처럼 보이는 화면 구성<br>스플래시 화면, 상태 바 색상, 전체 화면 모드 | PWA 실행 시 로고 표시 → 브라우저 주소창 숨김 → 앱 같은 느낌 |

## PWA의 한계 (플랫폼별 편차)

### iOS Safari 제약 사항

| 기능 | Android Chrome | iOS Safari | 비고 |
|------|---------------|------------|------|
| 홈화면 설치 | ✅ 지원 | ✅ 제한적 지원 | 별도 안내 필요 |
| 푸시 알림 | ✅ 완전 지원 | ⚠️ iOS 16.4+ 웹 푸시 제한적 지원 | PWA 설치 시에만 가능 |
| 백그라운드 동기화 | ✅ 지원 | ❌ 미지원 | |
| 기기 API 접근 | 광범위 | 제한적 | 카메라·위치 등 일부 제한 |
| 서비스 워커 캐시 | ✅ 완전 지원 | ✅ 지원 | iOS도 지원 but 용량 제한 |

### PWA 선택 전 반드시 확인해야 할 3가지

| 확인 항목 | 무슨 뜻인가요? | 왜 중요한가요? |
|----------|---------------|---------------|
| **1. 아이폰 푸시 알림** | 아이폰(iOS 16.4 이상)에서도 웹 푸시 알림을 보내려면, 사용자가 먼저 수동으로 **"홈화면에 추가"**를 눌러야 합니다 | 앱처럼 자동으로 알림 권한을 요청할 수 없습니다. 사용자가 직접 설정하지 않으면 푸시를 받을 수 없어 **도달률이 크게 낮아집니다** |
| **2. 캐시 자동 정리** | 아이폰 Safari는 한동안 사용하지 않은 PWA의 저장 데이터를 **자동으로 삭제**할 수 있습니다 | 오프라인 기능을 만들어도, 사용자가 며칠간 미방문 시 데이터가 사라져 **다시 로딩해야 합니다** |
| **3. 브라우저 업데이트 영향** | PWA는 Safari나 Chrome 브라우저 엔진에 의존하므로, 브라우저 업데이트 시 **동작이 달라질 수 있습니다** | 앱은 자체 업데이트로 제어하지만, PWA는 애플·구글의 브라우저 정책에 영향을 받습니다 |

> **결론**: PWA는 "**앱과 비슷한 경험을 가장 저렴하게**" 제공하는 대안입니다. 다만, **아이폰 사용자에게 푸시 알림이 핵심 서비스**라면 PWA만으로는 부족할 수 있어 앱 도입을 함께 검토해야 합니다.
