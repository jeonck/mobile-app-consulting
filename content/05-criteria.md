# 6. 앱 도입 필요성 판단 기준

## 6-1. 앱이 필요한 경우

아래 조건이 다수 겹칠수록 타당성이 높아집니다.

- 회원이 **반복적으로 방문**
- **푸시 알림**이 실질적인 재방문·참여 유도 수단
- **생체인증·간편로그인·기기기반 인증**의 편의가 중요
- **카메라·파일·위치·오프라인 저장** 등 기기 기능 연동이 서비스 가치에 직접 연결
- 앱 설치 후 **락인 효과**로 KPI(문의 감소, 전환율 상승, 재구매 증가) 확인

### 진입 마찰(Friction) 지표로 판단하기

| 지표 | 앱 필요 신호 | 웹 유지 신호 |
|------|-------------|-------------|
| 재방문 시 로그인 재인증 비율 | 30% 이상 | 10% 미만 |
| 모바일 평균 세션 길이 | 5분 이상 (깊은 이용) | 2분 미만 (단순 조회) |
| 홈화면 북마크/바로가기 이용률 | 낮음 (매번 URL 검색하거나 링크 찾아 헤맴) | 높음 (사용자가 이미 경로를 외워두고 자주 이용) |
| 이탈률(진입 페이지 이탈) | 40% 이상 | 20% 미만 |

> **업계 통계**: 앱 사용자의 평균 **체류 시간 및 PV가 웹 대비 2~3배 높습니다**.
> 이는 진입 마찰 감소가 직접적인 이용 깊이로 연결됨을 의미합니다.

## 6-2. 아직 앱이 이른 경우

- 서비스가 공지·조회·신청·주문 중심
- 사용 빈도가 낮음
- 푸시나 기기 연동이 핵심 아님
- 모바일웹 UX 개선만으로도 해결 가능한 문제

> **권고**: 이 경우 **모바일웹 고도화** 또는 **PWA 우선**이 비용 대비 효과가 높습니다.

## 6-3. 사용자 여정 관점에서의 판단

"**앱이 필요한가**?" 라는 질문은 다음과 같이 구체화해야 합니다:

1. 사용자가 우리 서비스에 **얼마나 자주** 방문하는가?
2. 방문 시마다 **로그인 마찰**을 겪고 있는가?
3. **푸시 알림**이 실제 행동(재방문, 참여)으로 이어지는가?
4. 모바일웹에서 **진입 경로**가 복잡한가?
5. 핵심 기능(카메라, 파일, 위치)이 서비스의 **핵심 가치**인가?

5가지 중 3개 이상이 "**Yes**"라면 앱 도입 타당성이 높습니다.
