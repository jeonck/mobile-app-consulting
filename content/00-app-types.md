# 모바일 앱의 이해: 개발 방식과 기술 유형

모바일 앱은 개발 방식과 기술에 따라 크게 **네이티브 앱**, **웹 앱**, **하이브리드 앱**으로 나뉘며, 최근에는 **크로스 플랫폼 앱**과 **PWA**(프로그레시브 웹 앱)도 많이 사용됩니다.

| 유형 | 핵심 키워드 |
|------|------------|
| **네이티브 앱** | 고성능 |
| **웹 앱** | 접근성 |
| **하이브리드 앱** | 효율성 |
| **크로스 플랫폼 앱** | 비용 절감 |
| **PWA** | 웹 기반 앱 경험 |

## 1. 네이티브 앱 (Native App)

| 항목 | 내용 |
|------|------|
| **특징** | iOS(Swift/Obj-C)나 Android(Kotlin/Java) 전용 언어로 개발하여 앱스토어에서 설치 |
| **장점** | 카메라, GPS 등 기기 기능 완벽 활용, 빠른 동작 속도 |
| **단점** | OS별 별도 개발 필요로 높은 비용/시간, 앱스토어 업데이트 필요 |

## 2. 웹 앱 (Web App)

| 항목 | 내용 |
|------|------|
| **특징** | 모바일 웹 브라우저에서 실행되는 웹사이트(HTML5, CSS, JS) |
| **장점** | 설치 불필요, 운영체제 상관없이 웹 주소로 즉시 접근 |
| **단점** | 기기 기능 활용 제한, 인터넷 연결 필수, 네이티브 대비 낮은 속도 |

## 3. 하이브리드 앱 (Hybrid App)

| 항목 | 내용 |
|------|------|
| **특징** | 웹 앱을 네이티브 앱 내에 감싸서(Wrapper) 앱스토어에 배포 |
| **장점** | 네이티브의 설치/기능 활용 + 웹의 빠른 업데이트/저렴한 비용 |
| **단점** | 네이티브 대비 UI/UX 성능이 미세하게 낮을 수 있음 |

## 4. 크로스 플랫폼 앱 (Cross-Platform App)

| 항목 | 내용 |
|------|------|
| **특징** | React Native, Flutter 등을 사용하여 하나의 코드로 iOS/Android 모두 지원 |
| **장점** | 개발 비용/시간 절감(네이티브 대비 약 50% 수준), 네이티브에 근접한 성능 |

## 5. 프로그레시브 웹 앱 (PWA)

| 항목 | 내용 |
|------|------|
| **특징** | 웹 기술로 개발되었으나, 네이티브 앱처럼 푸시 알림, 홈 화면 추가, 오프라인 사용이 가능한 앱 |

## 유형별 특징 요약

| 구분 | 네이티브 앱 | 하이브리드 앱 | 웹 앱 |
|------|------------|--------------|-------|
| **개발 언어** | Swift/Kotlin | 웹 언어(JS/HTML) + 네이티브 | 웹 언어(JS/HTML) |
| **설치방식** | App Store/Play Store | App Store/Play Store | 브라우저 접속 |
| **성능** | 최고 | 중간 | 낮음 |
| **비용/시간** | 높음 | 중간 | 낮음 |
| **기기 접근** | 자유로움 | 제한적 | 제한적 |

> **결론**: 높은 성능과 기능이 필요하면 **네이티브**, 개발 속도와 비용 절감이 중요하면 **하이브리드/크로스 플랫폼**, 단순한 콘텐츠 제공이 목적이면 **웹 앱**이 적합합니다.
