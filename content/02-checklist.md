# 2. 검토 항목 (내부 검토서 골격)

가장 먼저 정리해야 할 검토 항목입니다.

| 검토영역 | 반드시 확인할 질문 | 앱이 유리한 신호 | 모바일웹/PWA가 충분한 신호 |
|----------|-------------------|------------------|---------------------------|
| **서비스 목적** | 앱 도입의 1차 목적이 무엇인가? (브랜딩, 재방문, 로그인 편의, 알림, 거래 전환, 회원 락인 등) | 재방문 유도와 개인화 접점이 핵심 | 단순 조회·신청·주문 중심 |
| **사용자 이용행태** | 모바일 접속 비중은? 재방문 빈도는? 핵심 업무 반복 여부? | 주간/월간 반복 이용이 높음 | 필요 시 접속형, 비정기 이용 |
| **기능 필요성** | 푸시, 생체인증, 카메라/파일업로드, 위치, 오프라인, 홈화면 상주가 꼭 필요한가? | 앱 고유 기능 요구가 명확 | 웹/PWA로 대부분 대체 가능 |
| **로그인/보안** | 간편로그인·생체인증·민감정보 보호 수준을 얼마나 높여야 하는가? | 강한 인증 UX가 핵심 경쟁력 | 웹 로그인으로 충분 |
| **성능/품질** | 속도, 부드러움, 대용량 처리, 기기 연동이 중요한가? | 고빈도 사용 + 성능 민감 | 콘텐츠/신청형 서비스 |
| **운영 체계** | iOS/Android 배포·심사·버전관리·장애 대응 인력이 있는가? | 전담 운영 가능 | 웹 운영 인력 중심 |
| **비용 대비 효과** | 설치자 수, 이용 증대, 업무 효율, 문의 감소, 매출 증가 등 정량 효과를 만들 수 있는가? | KPI가 명확하고 회수 가능 | 효과가 모호함 |
| **대체수단** | 모바일웹 개선, PWA, 앱패키징으로 대체 가능한가? | 대체수단 한계가 명확 | 중간대안으로 충분 |
| **마케팅/소통** | 푸시 알림 도달률·CTR vs 기존 SMS/알림톡 효율성? 푸시 서버 운영비 대비 절감액은?<br>• 타겟팅 맞춤형 알림(개인화) 가능 여부 | 푸시 CTR > SMS 응답률, 알림톡 비용 절감 가능 | 기존 알림 채널로 충분 |
| **보안/규제** | 웹 브라우저 보안 한계(키로깅·화면캡처·세션하이재킹) vs 앱 내 보안 키패드·위변조 방지 필요성?<br>• 루팅/제일브레이킹 탐지, 보안 사고 대응 속도 | 금융·민감정보 처리, 위변조 방지 필수 | 웹 표준 보안 수준으로 충분 |
