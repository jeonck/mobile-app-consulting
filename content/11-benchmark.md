# 9. 벤치마킹 및 사례 연구

앱 도입 여부를 판단할 때 **유사 서비스의 실제 사례**를 참고하는 것은 의사결정권자를 설득하는 가장 강력한 도구입니다.

## 앱 도입 사례

| 기관/서비스 | 도입 배경 | 도입 결과 | 시사점 |
|------------|----------|----------|--------|
| **정부24 앱** | 민원 신청의 반복성·긴급성 높음 | 연 1,000만 다운로드, 푸시 기반 민원 처리 알림 활성화 | 고빈도·긴급성 서비스는 앱 효과적 |
| **카드사 앱** | 생체인증·실시간 알림·QR 결제 필요 | 앱 이용 비중 80% 이상, 웹은 단순 조회용 | 금융권은 보안+편의성 앱 필수 |
| **배달 플랫폼** | 위치 기반·실시간 추적·푸시 알림 핵심 | 앱 전환율 90% 이상, 웹은 신규 유입용 | 실시간성·위치 기반 서비스는 앱 우위 |

## 웹 유지 사례

| 기관/서비스 | 유지 배경 | 결과 | 시사점 |
|------------|----------|------|--------|
| **공공 포털** | 저빈도 이용, 정보 조회 중심 | 모바일웹으로 충분, 앱 유지보수 비용 대비 효과 미미 | 저빈도 서비스는 웹 충분 |
| **신문/미디어** | 콘텐츠 소비 중심, SEO 중요 | PWA로 홈화면 설치만 추가, 앱은 제한적 | 콘텐츠 중심은 PWA 대안 |
| **이커머스 소형** | 트래픽 대비 앱 설치율 낮음 | 웹 고도화 + PWA로 전환, 앱은 보류 | 설치율 20% 미만이면 앱 비효율 |

## 플랫폼 정책 리서치

### iOS vs Android PWA 지원 현황

| 항목 | Android | iOS |
|------|---------|-----|
| 웹 푸시 알림 | ✅ Chrome 완전 지원 | ⚠️ iOS 16.4+ 제한적 (PWA 설치 필요) |
| 홈화면 설치 | ✅ 자동 감지 | ⚠️ 수동 "공유 → 홈화면에 추가" |
| 백그라운드 동기화 | ✅ 지원 | ❌ 미지원 |
| 앱 이름·아이콘 설정 (매니페스트) | ✅ 완전 지원 | ✅ 지원 (일부 제한) |

> **핵심**: iOS의 PWA 지원은 개선되고 있으나, Android 대비 **1~2세대 뒤쳐져 있습니다**. iOS 사용자의 푸시 알림 활용도가 핵심이라면 앱 도입을 검토해야 합니다.

## 적용 가이드

### "우리 서비스에 맞는 채널은?"

```
1단계: 이용 빈도 확인
  ├─ 월 1회 미만 → 모바일웹 고도화
  ├─ 월 2~4회 → PWA 검토
  └─ 주 1회 이상 → 앱 도입 타당성 있음

2단계: 핵심 기능 확인
  ├─ 조회/신청 중심 → 웹/PWA
  ├─ 실시간 알림 필요 → 앱 또는 PWA (iOS 주의)
  └─ 기기 연동 필요 → 앱

3단계: 예산/역량 확인
  ├─ 제한적 → PWA → 앱 래핑 순
  └─ 충분 → 크로스플랫폼 MVP
```
